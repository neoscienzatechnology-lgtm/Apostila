Aula 2.4: Animações e Design Responsivo

Módulo 2 - CSS | Duração: 2 horas

Objetivos da Aula

  • Dominar transições e animações CSS
  • Implementar design responsivo eficaz
  • Otimizar performance de animações
  • Criar experiências interativas fluidas

1. Transições CSS

Transições permitem animar mudanças de propriedades CSS de forma suave.

Propriedades Principais

  • transition-property: O que será animado
  • transition-duration: Duração da transição
  • transition-timing-function: Curva de aceleração
  • transition-delay: Atraso antes de iniciar
.button {
    background: #3b82f6;
    transform: translateY(0);
    transition: all 0.3s ease;
}

.button:hover {
    background: #2563eb;
    transform: translateY(-2px);
}

/* Múltiplas propriedades */
.card {
    transition: 
        transform 0.3s ease,
        box-shadow 0.2s linear,
        opacity 0.2s ease-in;
}

2. Animações Keyframe

Definindo Keyframes

@keyframes fadeIn {
    from {
        opacity: 0;
        transform: translateY(20px);
    }
    to {
        opacity: 1;
        transform: translateY(0);
    }
}

@keyframes pulse {
    0% { transform: scale(1); }
    50% { transform: scale(1.05); }
    100% { transform: scale(1); }
}

.element {
    animation: fadeIn 0.5s ease forwards,
               pulse 2s ease infinite;
}

Propriedades de Animação

.element {
    animation-name: fadeIn;
    animation-duration: 1s;
    animation-timing-function: ease;
    animation-delay: 0.5s;
    animation-iteration-count: infinite;
    animation-direction: alternate;
    animation-fill-mode: forwards;
    animation-play-state: running;
    
    /* Shorthand */
    animation: fadeIn 1s ease 0.5s infinite alternate forwards;
}

3. Design Responsivo

Media Queries

/* Mobile First */
.container {
    width: 100%;
    padding: 1rem;
}

/* Tablet */
@media (min-width: 768px) {
    .container {
        max-width: 720px;
        padding: 2rem;
    }
}

/* Desktop */
@media (min-width: 1024px) {
    .container {
        max-width: 960px;
    }
}

/* Orientação */
@media (orientation: landscape) {
    .hero { height: 70vh; }
}

/* Preferências do usuário */
@media (prefers-reduced-motion: reduce) {
    * {
        animation-duration: 0.01ms !important;
        transition-duration: 0.01ms !important;
    }
}

Unidades Responsivas

:root {
    /* Base de 16px */
    font-size: 16px;
}

.title {
    font-size: 2rem;          /* Relativo ao root */
    margin-bottom: 1.5em;     /* Relativo ao próprio font-size */
}

.hero {
    height: 80vh;             /* % da altura da viewport */
    padding: clamp(1rem, 5vw, 3rem);  /* Responsivo com limites */
}

.grid {
    grid-template-columns: repeat(auto-fit, minmax(min(100%, 300px), 1fr));
}

4. Performance e Otimização

Propriedades para Animação

/* Bom - Usa GPU */
.good-animation {
    transform: translateX(100px);
    opacity: 0.5;
}

/* Ruim - Causa reflow */
.bad-animation {
    left: 100px;
    height: 200px;
}

/* Otimização */
.optimized {
    will-change: transform, opacity;
    transform: translateZ(0);  /* Force GPU */
}

Loading e Estados

.skeleton {
    background: linear-gradient(90deg,
        #f3f4f6 0%,
        #e5e7eb 50%,
        #f3f4f6 100%
    );
    background-size: 200% 100%;
    animation: loading 1.5s infinite;
}

@keyframes loading {
    from { background-position: 200% 0; }
    to { background-position: -200% 0; }
}

/* Estados de Loading */
.button {
    position: relative;
}

.button.loading::after {
    content: "";
    position: absolute;
    width: 20px;
    height: 20px;
    border: 2px solid #fff;
    border-right-color: transparent;
    border-radius: 50%;
    animation: spin 0.8s linear infinite;
}

5. Dicas Avançadas

Performance

  • Anime apenas transform e opacity quando possível
  • Use will-change com moderação
  • Evite animar muitos elementos simultaneamente
  • Teste performance em dispositivos reais

Acessibilidade

  • Respeite prefers-reduced-motion
  • Mantenha animações sutis e não intrusivas
  • Evite flashes e movimentos bruscos
  • Forneça alternativas para conteúdo animado

Boas Práticas

  • Use mobile-first em media queries
  • Teste em múltiplos dispositivos e browsers
  • Mantenha consistência nas animações
  • Documente breakpoints e timing functions
