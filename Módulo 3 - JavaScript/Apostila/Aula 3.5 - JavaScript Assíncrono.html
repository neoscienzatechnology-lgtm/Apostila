Aula 3.5: JavaScript Assíncrono

Callbacks, Promises, async/await, fetch e padrões para lidar com requisições e concorrência.

1. Callbacks e problemas clássicos

Callbacks são a forma original de tratar assincronismo em JS, mas podem gerar callback hell. Preferir Promises e async/await para clareza.

// Exemplo com callback
function lerArquivo(callback) {
  setTimeout(() => callback(null, 'conteúdo'), 1000);
}
lerArquivo((err, data) => { if (err) console.error(err); else console.log(data); });

2. Promises

Promises representam uma operação que será resolvida no futuro — permitem encadeamento e melhor tratamento de erros.

const p = new Promise((resolve, reject) => {
  // operação assíncrona
  setTimeout(() => resolve(42), 500);
});

p.then(v => console.log('valor', v))
 .catch(err => console.error('erro', err));

API: Promise.resolve, Promise.reject, Promise.all, Promise.race, Promise.allSettled.

3. async / await

async/await é sintaxe sobre Promises que deixa código assíncrono mais parecido com o síncrono; sempre trate erros com try/catch.

async function fetchUsers(){
  const res = await fetch('https://jsonplaceholder.typicode.com/users');
  if (!res.ok) throw new Error('Erro na requisição');
  return res.json();
}

async function run() {
  try {
    const users = await fetchUsers();
    console.log(users);
  } catch (err) {
    console.error(err);
  }
}
run();

4. Padrões de concorrência

// rodar várias requisições em paralelo
const urls = ['/u1','/u2','/u3'];
const promises = urls.map(u => fetch(u).then(r => r.json()));
const all = await Promise.all(promises); // falha se qualquer uma falhar

// Promise.allSettled para capturar todos os resultados
const settled = await Promise.allSettled(promises);

Escolha o padrão certo: paralelismo (all), corrida (race), ou tolerância a falhas (allSettled).

5. Fetch API — práticas e tratamento de erros

// fetch básico
fetch('/api/data')
  .then(res => {
    if (!res.ok) throw new Error('HTTP ' + res.status);
    return res.json();
  })
  .then(data => console.log(data))
  .catch(err => console.error('Falhou', err));

// cancelar usando AbortController
const ac = new AbortController();
fetch('/api/long', { signal: ac.signal }).catch(err => {
  if (err.name === 'AbortError') console.log('cancelado');
});
ac.abort();

6. Retry, backoff e política de erro

Requisições de rede podem falhar; implemente padrões de retry com exponencial backoff e limite máximo de tentativas.

async function retry(fn, attempts = 3, delay = 500) {
  let i = 0;
  while (i < attempts) {
    try {
      return await fn();
    } catch (err) {
      i++;
      if (i === attempts) throw err;
      await new Promise(r => setTimeout(r, delay * Math.pow(2, i)));
    }
  }
}

7. Streams e leitura progressiva (visão geral)

Fetch suporta streams via res.body.getReader() para processar dados progressivamente (útil para grandes downloads ou processamento em tempo real).
